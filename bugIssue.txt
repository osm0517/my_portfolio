그에 대한 공부와 내용 정리는 차후에 한 번에 정리하는걸로

1. cors에 대한 이슈
-> axios를 사용한 서버와 통신을 할 경우에 계속해서 응답 코드는 200번 이지만 에러가 발생
-> 발생하는 에러로는 cors 에러가 발생함
-> fetch를 사용하는 것으로 임시적으로 에러를 해결함
-> cors는 브라우저가 서버에 요청을 하고 cors에 위반한 요청이라고 판단되면 브라우저에서 버린다.
-> 위와 같은 특성때문에 서버와 서버 간에는 cors정책이 적용되지 않는다. 그리고 위반한 요청이라도 서버에서는 버그가 발생하지 않는다.

2. 원하지 않는 url로 get요청이 자꾸 보내지는 이슈
-> 클라이언트(현재 사이트)에 요청을 보내서 404에러가 발생함
-> http method를 사용할 때에 url을 다 작성하지 않고 /뒤쪽만 작성을 하면서 발생함

3. db 와 서버 간 통신을 할 때에 버그가 발생
-> ER_NOT_SUPPORTED_AUTH_MODE에러 발생
-> mysql ver8이 되면서 user plugin이 달라지면서 생기는 버그라고 함
-> localhost를 host로 가지는 root파일을 최신 버전에 plugin이 아닌 예전 버전으로 해서 만들어줌

4. react를 사용할 때에 이미지 경로 표시
-> 구글링을 통해서 확인한 내용은 public까지 경로는 자동으로 된다고 나와있지만 안됨
-> 일반적으로 경로를 설정하듯이 하면 인식을 하지 못함
-> 예상하건데 html이 아닌 html 문법을 사용할 수 있게 해주는 jsx여서 그런듯함
-> 이미지를 사용하는 방법은 총 2가지가 존재함
1. import를 사용해서 이미지를 사용하는 방법 -> 잘 바뀌지 않는 이미지를 사용할 때 사용함. 이유로는 굉장히 귀찮음
2. process.env.PUBLIC_URL를 이용해서 public폴더에 접근 후에 public 하위 폴더로 접근하여 사용하는 방법(현재 이 방법을 사용하도록 함)
